# Promote Idea — 아이디어 승격

idea-forge 세션에서 생성된 아이디어를 `idea-forge/ideas/{slug}/`로 승격하여 독립 관리한다.

## 입력

다음 중 하나를 받는다:
- **번호**: "1번 아이디어 승격" → 가장 최근 세션의 Top N에서 해당 번호
- **이름**: "vert-agent 승격" → 이름으로 검색
- **복수**: "1, 3번 승격" 또는 "전부 승격"
- **세션 지정**: "4월 5일 세션에서 2번 승격" → 특정 세션 지정

## 워크플로우

### Step 1: 세션 보고서 찾기

1. 세션이 지정되지 않으면 `idea-forge/sessions/`에서 가장 최근 디렉토리를 찾는다.
2. 해당 세션의 `final-report.md`를 Read로 읽는다.
3. 아이디어 스코어보드에서 사용자가 지정한 아이디어를 식별한다.

### Step 2: slug 결정

아이디어명에서 kebab-case slug를 생성한다.
- "VertAgent" → `vert-agent`
- "FoodLens KR" → `food-lens-kr`
- 이미 `idea-forge/ideas/{slug}/`가 존재하면 **재승격**(Step 4)으로 분기한다.

### Step 3: card.md 생성

`idea-forge/ideas/{slug}/card.md`를 생성한다. final-report.md에서 해당 아이디어의 데이터를 추출한다:

```markdown
# {아이디어명}

상태: active
승격일: {날짜}
출처 세션: sessions/{세션 디렉토리}/final-report.md
총점: {N}/50

## 원라이너
{한 문장 요약}

## 킬러 유즈케이스
{첫 경험 시나리오}

## 스코어링

| 기준 | 점수 | 가중치 | 근거 요약 |
|------|------|--------|----------|
| 트렌드 적합성 | {N}/5 | ×2 | {근거} |
| 시장 규모 | {N}/5 | ×2 | {근거} |
| 경쟁 우위 | {N}/5 | ×2 | {근거} |
| 실현 가능성 | {N}/5 | ×1.5 | {근거} |
| 수익성 | {N}/5 | ×1.5 | {근거} |
| 타이밍 | {N}/5 | ×1 | {근거} |

## 핵심 근거

### 트렌드
{이 아이디어와 직접 관련된 트렌드 데이터를 final-report에서 발췌}

### 시장
{관련 시장 규모, 경쟁, 타겟을 final-report에서 발췌}

## 수익 모델
{수익 모델 요약}

## MVP 범위 (4주)
{최소 버전 기능}

## 리스크

| 리스크 | 확률 | 영향 | 대응 |
|--------|------|------|------|

## 성장 경로
MVP → PMF → Scale
```

핵심 근거(트렌드 + 시장)를 반드시 포함한다. 세션 데이터가 cleanup된 후에도 심화에 필요한 맥락이 card.md 안에 살아있어야 한다.

### Step 4: 재승격 (이미 존재하는 아이디어)

이미 `idea-forge/ideas/{slug}/card.md`가 존재하면:
1. 기존 card.md 하단에 `## 이력` 섹션을 추가/append한다:
   ```markdown
   ## 이력
   - {날짜} (세션: {세션명}): 총점 {이전}/50 → {현재}/50. {주요 변화 요약}
   ```
2. 새 세션의 점수와 근거로 card.md의 스코어링 섹션을 업데이트한다.
3. 핵심 근거에 새 데이터를 병합한다 (기존 데이터 삭제하지 않음).

### Step 5: _index.md 업데이트

`idea-forge/ideas/_index.md`를 생성 또는 업데이트한다:

```markdown
# Idea Forge — 아이디어 인덱스

| 이름 | 상태 | 점수 | 승격일 | 파일 수 |
|------|------|------|--------|--------|
| [{이름}](./{slug}/) | active | {N}/50 | {날짜} | {N} |
```

### Step 6: 커밋

```
git add idea-forge/ideas/
git commit -m "idea-forge: promote {slug}"
```

### Step 7: 성장 안내

승격 완료 후 사용자에게 안내한다:

```
✓ {아이디어명}을 승격했습니다 → idea-forge/ideas/{slug}/card.md

다음 명령으로 아이디어를 성장시킬 수 있습니다:
- "심화해줘" → deep-dive.md (추가 리서치 + 경쟁 심층 분석)
- "검증 계획 세워줘" → validation.md (1주 검증 실험 설계)
- "피벗해줘" → pivot-log.md (방향 전환 + 이전 버전 보존)
- "구체화해줘" → spec.md (PRD 수준 상세 기획서)
```

## 성장 파일 생성

사용자가 승격된 아이디어에 대해 심화/검증/피벗/구체화를 요청하면, 해당 아이디어의 `card.md`를 Read로 읽고 요청에 맞는 파일을 생성한다:

| 트리거 | 파일 | 내용 |
|--------|------|------|
| "심화해줘" | `deep-dive.md` | WebSearch로 추가 리서치 + 경쟁사 심층 + 사용자 인터뷰 설계 |
| "검증 계획" | `validation.md` | 1주 검증 실험 설계 + 성공 기준 + 측정 지표 |
| "피벗해줘" | `pivot-log.md` | 피벗 이유 + 새 방향 + 이전 card.md 스냅샷 보존 |
| "구체화해줘" | `spec.md` | PRD 수준 상세 기획서 — 기능 목록, 와이어프레임 설명, 기술 스택, 일정 |

성장 파일 생성 후 `_index.md`의 파일 수를 업데이트하고 커밋한다.
