# Idea Generation — 프로젝트 아이디어 생성/평가/고도화

트렌드와 시장 데이터를 기반으로 프로젝트 아이디어를 생성하고, 구조화된 스코어링으로 평가하며, 고도화하는 스킬.

## 모드 판별

| 모드 | 조건 | 행동 |
|------|------|------|
| **생성 모드** | 사용자 아이디어 없음 | 트렌드×시장 교차점에서 아이디어 5~7개 생성 |
| **고도화 모드** | 사용자 아이디어 있음 | 평가 → 개선안 → 파생 아이디어 3~5개 |

## 생성 모드 워크플로우

### Step 1: 교차점 발굴

트렌드 보고서와 시장 분석 보고서를 읽고, 다음 매트릭스를 만든다:

```
        시장 기회 A    시장 기회 B    시장 기회 C
트렌드 1    ✦ 강한 교차    · 약함        · 없음
트렌드 2    · 약함        ✦ 강한 교차    ◆ 중간
트렌드 3    ◆ 중간        · 없음        ✦ 강한 교차
```

`✦`로 표시된 강한 교차점을 우선 아이디어 후보로 선정한다.

### Step 2: 아이디어 구체화

각 교차점에서 다음을 정의한다:

1. **원라이너**: 한 문장으로 아이디어 요약 ("{타겟}을 위한 {솔루션}으로 {가치}를 제공")
2. **킬러 유즈케이스**: 사용자가 "이거다!"라고 느끼는 첫 경험
3. **작동 원리**: 어떻게 작동하는지 3단계로 설명
4. **차별화**: 기존 솔루션 대비 10x 개선점
5. **수익 모델**: 어떻게 돈을 버는가
6. **MVP 범위**: 4주 안에 만들 수 있는 최소 버전

### Step 3: 아이디어 평가 (스코어링)

각 아이디어를 다음 루브릭으로 1~5점 평가한다:

| 기준 | 1점 | 3점 | 5점 | 가중치 |
|------|-----|-----|-----|--------|
| **트렌드 적합성** | 트렌드와 무관 | 트렌드와 간접 연결 | 핵심 트렌드 정중앙 | ×2 |
| **시장 규모** | SOM < 10억 | SOM 10~100억 | SOM > 100억 | ×2 |
| **경쟁 우위** | 차별화 없음 | 일부 차별화 | 10x 개선점 명확 | ×2 |
| **실현 가능성** | 대규모 팀/자본 필요 | 소규모 팀 가능 | 1인 개발 가능 | ×1.5 |
| **수익성** | 수익 모델 불명확 | 수익 모델 있으나 검증 필요 | 검증된 수익 모델 | ×1.5 |
| **타이밍** | 너무 이르거나 늦음 | 적절한 시기 | 지금이 아니면 안 됨 | ×1 |

**총점** = 각 (점수 × 가중치)의 합. 최고 50점.

### Step 4: 순위화 및 Top 3 심화

총점 기준 상위 3개 아이디어를 선정하고, 각각에 대해:

- **리스크 분석**: 실패할 수 있는 3가지 이유와 대응 방안
- **성장 경로**: MVP → Product-Market Fit → 스케일링 로드맵
- **빠른 검증 방법**: 1주일 안에 이 아이디어를 검증하는 방법

## 고도화 모드 워크플로우

### Step 1: 사용자 아이디어 분석

사용자 아이디어를 위의 스코어링 루브릭으로 평가한다. 각 기준에 대해 점수와 근거를 제시한다.

### Step 2: 개선안 제시

낮은 점수를 받은 기준에 대해 구체적 개선 방향을 제시한다:

- "경쟁 우위 2점 → 5점으로 올리려면: {구체적 피벗/기능 추가/포지셔닝 변경}"
- 개선안 적용 후 예상 점수를 재계산한다.

### Step 3: 파생 아이디어 생성

사용자 아이디어의 핵심 가치를 유지하면서 다른 형태로 변환한다:

| 변환 축 | 예시 |
|---------|------|
| 타겟 변경 | B2C → B2B, 젊은층 → 시니어 |
| 채널 변경 | 앱 → 오프라인, 구독 → 마켓플레이스 |
| 가치 확장 | 정보 제공 → 커뮤니티 → 커머스 |
| 기술 레버리지 | 수동 → AI 자동화, 단독 → 플랫폼 |
| 니치 집중 | 범용 → 특정 버티컬 (반려동물, 육아, 투자 등) |

각 파생 아이디어도 동일한 스코어링으로 평가한다.
