# Research Topic — 발표 주제 리서치

발표 주제에 대한 심층 리서치를 수행하여 콘텐츠의 근거를 마련한다. 두 가지 모드를 지원한다:
- **신규 조사 모드**: `_workspace/00_original_script.md`가 없을 때. 아래 "리서치 프로세스"를 따른다.
- **검증/보강 모드**: `_workspace/00_original_script.md`가 있을 때. 아래 "검증/보강 모드"를 따른다.

## 리서치 프로세스

### 1단계: 주제 분해
주제를 다음 관점으로 분해한다:
- **What**: 핵심 개념, 정의, 범위
- **Why**: 왜 중요한가, 어떤 문제를 해결하는가
- **How**: 어떻게 작동/적용하는가
- **Evidence**: 뒷받침하는 데이터, 사례, 연구
- **Trend**: 최신 동향, 미래 전망

### 2단계: 멀티 소스 조사
다양한 출처에서 정보를 수집한다:

| 소스 유형 | 검색 전략 | 신뢰도 |
|----------|----------|--------|
| 공식 문서/논문 | 학술 키워드 + "paper" "research" | 높음 |
| 기업 블로그/기술 문서 | 기업명 + 주제 | 중간-높음 |
| 뉴스/미디어 | 최신 날짜 필터 + 주제 | 중간 |
| 커뮤니티/SNS | 실무자 관점, 실제 경험 | 맥락 의존 |

WebSearch로 최소 5회 이상 검색하되, 키워드를 변형하여 다양한 관점을 수집한다. 하나의 키워드에 의존하면 편향된 정보를 얻는다.

### 3단계: 정보 정리

수집한 정보를 발표에 활용 가능한 형태로 정리한다:

- **핵심 수치**: "X는 Y% 증가" 형태로 한 줄 요약
- **인용문**: 권위 있는 출처의 인용 가능한 문장
- **사례**: 3문장 이내로 압축한 성공/실패 사례
- **비교 데이터**: 차트로 만들 수 있는 수치 세트

## 출력 형식

```markdown
# 리서치 결과: [주제]

## 핵심 개념
[정의와 범위]

## 핵심 통계/데이터
- [출처] [수치와 맥락]
- ...

## 주요 사례
### 사례 1: [제목]
[3문장 요약]
출처: [URL]

## 최신 트렌드
[동향 요약]

## 인용 가능 문장
- "[인용문]" — [출처, 날짜]

## 시각화 가능 데이터
[차트로 만들 수 있는 수치 세트, 비교 데이터]

## 출처 목록
1. [제목] - [URL] - [날짜] - [신뢰도: 높음/중간]
```

## 검증/보강 모드

`_workspace/00_original_script.md`에 기존 대본이 있으면 이 모드로 동작한다. 새 주제를 탐색하는 게 아니라, 기존 대본에 인용된 주장과 데이터의 정확성을 검증하고 더 강력한 근거를 찾는다.

### 1단계: 기존 대본에서 검증 대상 추출

대본을 읽고 다음을 목록화한다:
- **인용된 수치**: "X는 Y%", "N만 줄의 코드" 등 구체적 숫자
- **인용된 사례**: 기업명, 프로젝트명, 인물명이 언급된 사례
- **인용된 출처**: 논문, 블로그, 보고서 등 명시적 출처
- **암묵적 주장**: 출처 없이 사실로 제시된 주장 ("업계 표준으로 자리잡았다" 등)

### 2단계: 항목별 검증

각 항목을 WebSearch로 검증한다:

| 검증 결과 | 처리 |
|----------|------|
| **정확함** | 출처 URL을 보강하여 기록 |
| **부정확/오래됨** | 최신 수치를 찾아 대체 데이터 제안 |
| **출처 없음** | 해당 주장을 뒷받침하는 출처 탐색. 못 찾으면 "미검증" 표시 |
| **더 강력한 대안 존재** | 원본 수치와 대안을 병기하고 추천 표시 |

### 3단계: 보강 자료 탐색

대본의 주장을 더 강화할 수 있는 추가 자료를 탐색한다:
- 더 최신의 데이터 (원본이 2024년이면 2025-2026 데이터 탐색)
- 더 권위 있는 출처 (블로그 → 공식 발표, 커뮤니티 → 논문)
- 대본에 없지만 추가하면 설득력이 올라갈 사례나 데이터

### 검증/보강 모드 출력 형식

```markdown
# 검증/보강 결과: [대본 제목]

## 검증 요약
- 검증 항목 수: N개
- 정확: N개 / 부정확·오래됨: N개 / 미검증: N개

## 항목별 검증 결과

### [대본 슬라이드 번호]: "[인용 원문]"
- 검증 결과: 정확 | 부정확 | 오래됨 | 미검증
- 원본 수치: [대본에 쓰인 값]
- 최신 수치: [검증된 값] (출처: [URL], 날짜: [날짜])
- 권장: 유지 | 수정 → "[대체 표현]"

## 보강 제안
### 추가 가능 데이터
- [슬라이드 N 근처에 추가 가능] [데이터 설명] (출처: [URL])

### 대체 가능 사례
- [기존 사례] → [더 강력한 대안 사례] (이유: [왜 더 나은지])

## 출처 목록
1. [제목] - [URL] - [날짜] - [신뢰도]
```

## 품질 기준
- 출처 없는 주장은 포함하지 않는다
- 2년 이상 된 통계는 "[주의: 날짜]" 표시
- 상충되는 정보는 양쪽 모두 보고
- 최소 3개 이상의 독립적 출처 확보
