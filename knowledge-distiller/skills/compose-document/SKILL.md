# Compose Document — 저작물 합성

영구 노트를 대상 저작물에 매핑하고 통합하여 보강된 초안을 생성한다.

## 워크플로우

### 1. 저작물 구조 분석

`_workspace/00_input/target-snapshot.md`를 읽고 분석한다:

- **전체 구조**: 섹션/장/슬라이드 구성
- **논리 흐름**: 현재 논리 전개 방향
- **톤/스타일**: 문체, 청중 수준, 형식성
- **기존 근거**: 이미 사용된 데이터/사례/인용
- **공백**: 주장은 있지만 근거가 부족한 부분

### 2. 통찰-저작물 매핑

`_workspace/permanents/` 내 영구 노트들과 관계 맵을 읽고, 각 통찰이 저작물의 어디에 배치될 수 있는지 매핑한다.

`_workspace/synthesis/flow-map-R{round}.md`에 매핑 결과를 저장한다:

```markdown
# Flow Map — 통찰 배치 계획

## 대상 저작물 구조 분석
{구조 요약}

## 매핑

### 섹션/슬라이드: {이름}
- **현재 내용**: {요약}
- **배치할 통찰**:
  - insight-{id}: {통찰 요약} → {어떻게 활용할 것인가}
  - insight-{id}: {통찰 요약} → {어떻게 활용할 것인가}
- **보강 방법**: 데이터 추가 / 사례 삽입 / 주장 강화 / 반론 보완

### 섹션/슬라이드: {이름}
...

## 제외된 통찰
- insight-{id}: {제외 이유}

## 발견된 공백
- {저작물에서 논리적으로 필요하지만 통찰이 부족한 부분}
- → 추가 리서치 주제 제안: {주제}
```

### 3. 저작물 보강

매핑에 따라 대상 저작물에 통찰을 통합한다.

**통합 규칙:**

1. **원작 존중**: 기존 내용의 삭제나 대폭 수정 없이 보강한다
2. **맥락 연결**: 통찰을 삽입할 때 전후 문맥과 자연스럽게 연결한다
3. **톤 일치**: 저작물의 기존 문체와 어조를 유지한다
4. **출처 추적**: 보강한 부분에 `<!-- [KD-R{round}] insight-{id} -->` 주석을 삽입한다
5. **선택적 통합**: 모든 통찰을 넣지 않는다 — 저작물의 목적과 흐름에 부합하는 것만 선택한다

보강된 초안을 `_workspace/output/enriched-draft-R{round}.md`에 저장한다.

### 4. 변경 요약 작성

`_workspace/output/changelog-R{round}.md`에 이번 라운드의 변경 사항을 기록한다:

```markdown
# 라운드 {N} 변경 요약

## 보강된 부분
| 위치 | 통찰 | 변경 유형 | 설명 |
|------|------|----------|------|
| 섹션 X | insight-001 | 데이터 추가 | SWE-bench 최신 수치로 업데이트 |
| 슬라이드 15 | insight-003 | 사례 삽입 | Manus 사례 추가 |

## 제외한 통찰과 이유
...

## 발견된 공백 (다음 라운드 리서치 후보)
...
```

### 5. 사용자 확인 포인트

보강된 초안을 사용자에게 제시하기 전에 확인할 사항:
- 원작의 핵심 메시지가 변경되지 않았는가
- 보강된 내용이 저작물의 분량에 적절한가 (과도한 삽입 방지)
- 논리 흐름이 자연스러운가 (끊어지는 지점이 없는가)
