# Distill Insights — 통찰 증류

참조 노트들을 교차 분석하여 원자적 통찰(영구 노트)을 추출한다.

## 워크플로우

### 1. 참조 노트 수집

`_workspace/references/` 내 모든 참조 노트를 읽는다. 기존 `_workspace/permanents/` 내 영구 노트도 읽어 중복을 방지한다.

### 2. 교차 분석

참조 노트들을 다음 렌즈로 분석한다:

| 렌즈 | 질문 |
|------|------|
| **패턴** | 여러 출처에서 반복적으로 등장하는 주장이나 현상은? |
| **연결** | 서로 다른 주제의 정보가 예상치 못하게 연결되는 지점은? |
| **모순** | 출처 간 상충하는 주장이 있는가? 왜 다른가? |
| **공백** | 논의되지 않았지만 논리적으로 필요한 빠진 조각은? |
| **전환점** | "이것이 사실이라면 기존 이해가 바뀌어야 하는" 발견은? |

### 3. 영구 노트 작성

발견한 각 통찰을 아래 템플릿으로 `_workspace/permanents/insight-{id}-{slug}.md`에 저장한다.

`{id}`는 3자리 숫자 (001부터). 기존 영구 노트가 있으면 그 다음 번호부터 시작한다.

```markdown
# Insight {id}: {통찰 제목}

- **확신도**: 🟢 강함 / 🟡 보통 / 🔴 약함
- **유형**: 패턴 / 연결 / 모순 / 공백 / 전환점
- **라운드**: {round}

## 통찰

{하나의 원자적 통찰 — 2~4문장으로 명확하게 기술}

## 근거

{이 통찰을 도출한 참조 노트와 해당 발견 인용}

- 출처 1: `references/{파일명}` — "{관련 발견 요약}"
- 출처 2: `references/{파일명}` — "{관련 발견 요약}"

## 의미

{이 통찰이 대상 저작물에 어떤 의미를 갖는가. "그래서 이것이 중요한 이유는..."}

## 관계

- **지지**: insight-{id} — {어떻게 지지하는가}
- **모순**: insight-{id} — {어떻게 모순되는가}
- **확장**: insight-{id} — {어떻게 확장하는가}
(관계가 없으면 "없음"으로 표시)
```

### 4. 관계 맵 업데이트

모든 영구 노트 작성 후, `_workspace/permanents/_relation-map.md`에 통찰 간 관계를 요약한다:

```markdown
# 통찰 관계 맵

## 클러스터

### 클러스터 A: {테마}
- insight-001 ← 지지 → insight-003
- insight-001 ← 확장 → insight-005

### 클러스터 B: {테마}
- insight-002 ← 모순 → insight-004

## 고립된 통찰
- insight-006 (관계 없음 — 추가 리서치 후보)

## 다음 라운드 제안 주제
- {공백이나 고립 통찰에서 도출된 추가 리서치 주제}
```

### 5. 품질 기준

- 각 영구 노트는 **하나의** 통찰만 담는다 (원자성)
- 통찰은 "X는 Y이다" 형태의 주장이어야 한다 (질문이나 메모가 아님)
- 참조 노트의 단순 요약이 아니라 **교차 분석의 결과**여야 한다
- 하나의 참조 노트에서만 도출된 통찰은 확신도 🔴로 표시한다
