# Research Sources — 참조 노트 작성

주어진 주제에 대해 다각적으로 조사하여 Zettelkasten 참조 노트를 작성한다.

## 워크플로우

### 1. 주제 분석

입력에서 다음을 파악한다:
- **주제**: 무엇을 조사할 것인가
- **조사 방향**: 어떤 측면에 집중할 것인가 (있으면)
- **대상 저작물 맥락**: 이 리서치가 어디에 사용될 것인가

### 2. 조사 수행

다음 순서로 조사한다:

1. **핵심 개념 조사** — WebSearch로 주제의 정의, 역사, 핵심 원리 파악
2. **최신 동향** — WebSearch로 최근 1년 내 트렌드, 발표, 변화 조사
3. **데이터/통계** — 정량적 근거가 될 수치, 벤치마크, 연구 결과 수집
4. **사례/실무** — 실제 적용 사례, 성공/실패 스토리 조사
5. **반론/한계** — 주제에 대한 비판, 한계, 주의점 조사

각 조사 시 WebSearch → WebFetch(유망한 출처) 패턴을 사용한다. 최소 3개 이상의 독립적 출처를 확보한다.

### 3. 참조 노트 작성

조사 결과를 아래 템플릿으로 정리하여 `_workspace/references/{round}-{topic-slug}.md`에 저장한다.

```markdown
# {주제명}

- **조사일**: {날짜}
- **라운드**: {round 번호}
- **조사 방향**: {조사의 초점}

## 핵심 개념

{주제의 핵심 정의, 원리, 배경 — 자신의 말로 재구성}

## 주요 발견

### 발견 1: {제목}
- **내용**: {핵심 내용}
- **출처**: {URL/저자/발행일}
- **중요도**: 🟢높음 / 🟡보통 / 🔴낮음
- **시사점**: {대상 저작물에 어떻게 활용 가능한가}

### 발견 2: {제목}
...

## 데이터/통계

| 지표 | 수치 | 출처 | 날짜 | 비고 |
|------|------|------|------|------|
| ... | ... | ... | ... | ... |

## 사례

### 사례 1: {제목}
- **배경**: ...
- **결과**: ...
- **교훈**: ...

## 반론/한계

{이 주제에 대한 비판적 시각, 한계, 주의점}

## 후속 조사 제안

{추가로 조사하면 좋을 세부 주제나 질문}
```

### 4. 품질 자가 점검

작성 완료 후 스스로 점검한다:
- [ ] 최소 3개 이상의 독립 출처가 있는가
- [ ] 모든 수치/통계에 출처와 날짜가 있는가
- [ ] 단순 복사가 아니라 재구성되어 있는가
- [ ] "시사점"이 대상 저작물 맥락에 맞게 작성되어 있는가
