# knowledge-distiller

Zettelkasten 방식의 지식 증류 하네스. 주제를 리서치하고, 참조 노트에서 핵심 통찰을 추출하여, 대상 저작물에 체계적으로 통합한다.

## 특징

- **Zettelkasten 3단계 노트 시스템**: 참조 노트(reference) → 영구 노트(permanent) → 저작물(composition)
- **멀티 라운드 축적**: 라운드마다 지식이 누적되어 통찰이 심화됨
- **교차 분석**: 서로 다른 주제의 정보가 결합되어 비자명한 통찰 발견
- **대상 인식**: 저작물의 구조와 공백을 분석하여 방향성 있는 리서치 수행
- **논리 흐름 매핑**: 통찰을 저작물의 흐름에 맞게 배치하는 합성 단계 포함

## 워크플로우

```
[주제 입력] → [리서치 팬아웃] → [통찰 증류] → [저작물 합성] → [결과 보고]
                                                                    ↓
                                                            [다음 라운드?]
                                                                    ↓
                                                            [Phase 1로 반복]
```

## 에이전트

| 에이전트 | 역할 |
|---------|------|
| researcher | 주제별 심층 리서치 → 참조 노트 작성 |
| distiller | 참조 노트 교차 분석 → 영구 노트(통찰) 추출 |
| composer | 통찰을 대상 저작물에 매핑 및 통합 |

## 사용 예시

```
# 대상 저작물과 함께
"Context Engineering과 Memory Engineering을 조사해서 이 발표 대본을 보강해줘"

# 통찰 추출만
"Harness Engineering 최신 동향을 조사하고 핵심 통찰을 정리해줘"

# 멀티 라운드
"이전 라운드에서 발견된 공백을 추가 조사해줘"
```
